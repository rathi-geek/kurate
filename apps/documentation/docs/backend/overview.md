# Backend Application

## Overview

This section contains documentation for backend applications in the monorepo.

## Getting Started

When you add a backend application (e.g., NestJS, Express, FastAPI), its documentation should be organized here following the standard structure:

- `overview.md` - Application architecture and purpose
- `setup.md` - Installation and configuration guide
- `development.md` - Development workflow and guidelines
- `api-reference.md` - API endpoints and documentation
- `troubleshooting.md` - Common issues and solutions

## Documentation Standards

All backend documentation must follow:

- **ISO Markdown Format**: Proper heading hierarchy, blank lines around lists
- **Clear Structure**: Consistent organization across all backend apps
- **Code Examples**: Include practical examples with language specification
- **API Documentation**: Complete endpoint documentation with examples

## Next Steps

1. Add your backend application to the `apps/` directory
2. Create documentation following the structure above
3. Update this overview with specific backend information
4. Link to your app-specific documentation
